Stack Overflow 2018 Developer Survey

The survey had over 100,000 developers answer questions about everything from their favorite technologies to their job preferences. What can it tell us about the current trends in programming languages?

About the Visualization
Hybrid Structure
This narrative visualization follows the martini glass structure that uses a hybrid of author-driven content followed by reader-driven content. The first four scenes both use annotations to point out observations and teach the user the navigations of the visualization one at a time. Finally the last scene enables all the filtering options and prompts the user to explore and identify their own trends.

Scenes
The visualization is presented as five scenes, each numbered and navigable as slides. The user can advance or go back with forward/previous buttons as well as navigate to any slide arbitrarily. All the scenes are laid out similarly with pagination, filters, text, and plot in the same location. When navigating from scene to scene, the plot transitions with bars growing and shrinking. The changed parameters flash subtly to cue the new values. The text narrative can be found in a summary box on the bottom left and with annotations that transition in after a delay to ease the user visually. Each scene resets all the parameters and annotations as needed so they work in both the ordered mode as well as when arbitrarily navigated to.

Annotations
In addition to a summary of the data presented in a textbox for each scene, specific regions of the plots are highlighted by using annotations. The annotation consists of text, a line, and when applicable a circle around the highlighted region. The annotations are all stylistically similar, i.e. font size, text length, color, and size. The annotations fade in with delays to draw attention and fade out when the scene is changed.

Parameters
  • Country is used to filter the survey respondents by their country of residence.
  • Age is used to filter the survey respondents by their age.
  • Years Coding is used to filter the survey respondents by the number of years they have been coding.
These parameters can be manipulated simultaneously to any combination on the fifth scene. In addition, the page number from 1 to 4 specifies the state of the visualization and results in pre-defined values for the other parameters. When any of these parameters are changed, the scene is updated to represent the new data.

Triggers
There are two types of triggers: scene navigation and parameter selection. The next/previous buttons and the 1-5 page buttons all change the scene that is currently displayed. Clicking (onclick event) any of these buttons sets the parameters to pre-defined values and updates the plot, annotations, and text narrative to match. On page 5 the drop down menus corresponding to the other parameters are enabled and changing (change event) them triggers the visualization to update with results matching the filters. The available controls are made known to the user by styling disabled options as greyed out and not interactable, while the active ones are colored and clickable. The flashes of the dropdown during the author-driven portion of the martini glass structure also draw attention to the drop down menus that are later enabled.
